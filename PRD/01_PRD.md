## 1. 제품 개요

### 한 줄 요약
CLI 스크립트로만 가능했던 QA 보고서 생성을 브라우저에서 누구나 실행할 수 있도록 Flask 기반 로컬 웹 UI로 전환한다.

### 해결하는 문제
QA 2인 팀이 보고서를 생성하려면 현재 CLI 환경, Python, .env 설정 지식이 필요하다. 기술 친화도가 다른 팀원도 브라우저만으로 리포트를 실행·게시할 수 있어야 한다.

### 핵심 가치
- 브라우저 폼에서 파라미터 입력 → 실행 → 실시간 로그 확인 → Confluence 링크 바로 열기까지 원클릭
- 기존 `core/` 로직 그대로 재사용, 새 언어나 프레임워크 추가 없음

## 2. 사용자

### 주요 사용자
- **누구**: Bodoc QA 팀 2인 (Jamie 외 1인)
- **상황**: 매월 MOR, 주기적 QA 보고서·테스트 플랜·연간 보고서 생성 시
- **목표**: Jira 티켓 키 또는 날짜 입력 → Confluence 페이지 자동 생성

### 사용자 시나리오
1. QA 팀원이 `python app.py` 실행 후 브라우저로 접속한다.
2. 탭에서 원하는 리포트 종류를 선택하고 파라미터(태스크 키, 이메일 등)를 입력한다.
3. "실행" 버튼을 누르면 실시간 로그가 스트리밍되고, 완료 시 Confluence 링크가 표시된다.

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
앱 실행 → 브라우저 접속 → 리포트 탭 선택 → 파라미터 입력 → 실행 → 로그 확인 → 결과 링크 열기
```

### 상세 흐름 (MOR 예시)
1. **탭 선택**: "MOR 리포트" 탭 클릭
2. **입력**: 월(2026-04), 이메일(jamieqa0@gmail.com), Confluence 게시 토글 ON
3. **실행**: 실행 버튼 → 로그 스트리밍 시작
4. **완료**: 성공 메시지 + Confluence 페이지 링크 표시 → 클릭해서 바로 확인

### 설정 흐름
1. 헤더의 "설정" 링크 클릭
2. Atlassian URL, API 토큰, Space Key, 부모 페이지 ID 입력
3. "저장" → `config/ui_settings.json`에 기록
